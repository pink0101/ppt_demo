# Thoughts on low code ## 关于低代码的思考 ###### 第十组 申超 钟粤 ###### 2022-05-15
## In the last century, to be a programmer, you had to know assembly language. Today, programming is almost universal. Most people can learn how to write a simple JavaScript script for a web page in a day
### 上个世纪要当程序员，你必须懂汇编语言。到了如今，编程几乎已经没有门槛了，大多数人一天内就能学会，怎么写一个简单的网页 JavaScript 脚本。那么，你告诉我，未来会怎样？
## There will still be programmers, but there will be very few of them, and they will only write the low-level code, just as there are very few assembly language programmers today. So I doubt that kids will have to learn to program, because the future may not be the way it is today
### 那时还会有程序员，但是人数应该很少，而且只负责编写底层代码，就像今天的汇编语言程序员非常少一样。因此我怀疑，孩子们没必要专门去学编程，因为未来可能根本不是今天这样编程。
## I think this is the trend of future programming tools. You don't need or only need to understand a little code to make software and solve your problems. The recent emergence of "low code" and "no code" tools coincides with this trend. In the future, everyone will be a software engineer and will be able to make the software they need, but almost no one knows how to program.
### 我觉得，这就是未来编程工具的趋势，你不需要或者只需要懂一点点代码，就能做出软件，解决你的问题。最近正在兴起的"低代码"和"无代码"工具，正好呼应了这种趋势。未来人人都是软件工程师，都能够做出自己需要的软件，但是几乎没人知道如何编程。
